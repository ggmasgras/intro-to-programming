1. Major Pieces of the Web

  • you & your computer + browser (IE, Chrome, Firefox, Safari, etc.)

Your computer browser makes a request over the internet to the servers using the HTTP protocol, and the servers respond back via the internet. Servers are like computers, just optimized for sitting in a closet, hosting files.

2. HTML

HTML stands for Hypertext Markup Language and represents the basis of almost every web page. HTML is what glues everything together and HTML documents are the "heart" of the web.

HTML is made up of:

  • text content — what you see
  • markup — what is looks like
  • references to other documents — e.g. images & video
  • links to other pages

HTML Attributes

Anchors

An anchor is used for creating links. They include:

  • href — attribute name
  • link — the value (i.e. www.reddit.com )

Image tags

An image tag is used to embed images on the web page. The tag includes:

  • img opening/closing tags
  • source (src)
  • alt (alternate) - text that displays when the image doesn't load — REQUIRED

Image tags are called void tags and are inline elements; they have no content and thus require no closing tag. Break is also a void tag, used for line breaks.

The break tag is an inline tag; it is ending a line ("the line ends here"). Other inline elements are the bold tag, and the emphasis tag.

The paragraph tag is NOT a void tag; it creates a box around the content so it needs a closing tag. The paragraph p tag is a block as well, as it creates an 'invisible' box around the text. This box can have height and width--which can be customized, as opposed to the break tag, for example, that only ends the line.

Boxes & Containers

Elemens such as Span & Div are used to create boxes and containers on an HTML page.

Span is an inline element creating an inline container element.

Div is a block element creating a 'box' container element. These elements do not do anything other than contain their content. There is a way to attach styling elements to them to influence the way in which they are displaying.

HTML Document

An HTML document include the following elements:

  • doctype
  • HTML tag - encompasses all the content on the page
  • Head tag - includes info such as metadata, Java script, CSS, as well as the title tag with the title of the page
  • Body tag - includes the content of the document

3. CSS

CSS stands for Cascading Style Sheets and is a stylesheet language used for describing the look and formatting of a document written in a markup language.

CSS is a collection of rules in which order matters. A key concept of CSS is most specific rule applied.

Order Matters

Styles can be defined in different places and are applied in the following order, with definitions further down the list overwriting previous definitions:

  • the detault style of the browser (different browsers have different styles)
  • stylesheet in a separate file (most common)
  • stylesheet in inside HTML (can be done for small projects, but not ideal)
  • inline style in an element (should be avoided)

Specifics Matter

Cascading means that rules are applied not only to the elements they directly match, but also to all of those elements' child elements. However, if a child element has multiple, overlapping rules defined for it, the more specific rule takes effect.

CSS Selectors

See full article at css-tricks.com/how-css-selectors-work

ID Selector

ID selectors are the most powerful type of selector in terms of CSS specificity. They outweigh other types of selectors and the styles defined within override other styles.

However, keep in mind that it is better to have lower-specificity selectors that are easier to override when needed.

Class Selector

Class selectors are probably the most useful and versatile selectors, and they are well supported in all browsers. You can also add multiple classes (just separated by a space) on HTML elements. There are also Javascript commands you can implement to specifically manipulate classes.

Tag Selectors

Tag selectors are most useful when changing properties that are unique to that HTML element. Examples include setting the 'list-style' on a ul element or 'tab-size' on a pre.

However, it is tipically more useful to have a class define styling that you can use on any HTML element.

Attribute Selector

Attribute selectors are arguably more useful than classes because they have the same specificity value, but can be any attribute, not just 'class,' plus they can have a value you can select by.

4. Developer Tools & The Box Model Design

All web pages can be broken down into boxes, from the biggest box containing the whole page to smaller boxes within containing the different elements of the page.

Chrome Developer Tools (DevTools) are a set of web authoring and debugging tools built into Chrome. Using DevTools we can easily explore the Box Model Design that stands at the core of front end web design.

Defining Boxes in HTML

Div Tags

Div tags are used in order to define individual boxes. Since boxes can contain other boxes, hence div tags can contain other div tags.

Div tags can be given a 'class' attribute, which functions like a label on a box, and can be used to define the style of the box.

A key concept to keep in mind is that boxes can have multiple labels.

Box Sizing

HTML elements are boxes and each element has 4 components ilustrated below:

In the center of the box, we have the content (pink), which can contain text or an image. The area surrounding the content is the padding (blue), which clears the area around the content. Next, we have the border (green), which goes around the area of the padding and the content, and it's inherited from the color property of the box. Lastly, we have the margin (grey), which clears the area around the border; the margin does not have a color and is completely transparent (space between boxes).

By default, the size of your element is equal to the sum of the border, padding and content width. This makes it a bit difficult to determine the size of an element, especially if we continue to change its parameters.

There are 2 techniques you can use to deal with sizing issues:

  • Set sizes in terms of percentages (%) rather than pixels
  • Use the CSS style attribute box-sizing:border box that makes the calculation of the element size including the border and padding, making it easier to make layouts. Using this style, you can just set the width of the overall box, so no matter how much you tweak the padding or border, the size of the box will always stay the same.

One drawback with the box model is that having been developed more recently, it is not necesarily compatible with all browsers. This means you need to add browser-specific prefixes so that older browsers are still able to display the element correctly. The prefixes are: -webkit, -moz, and -ms, and they can be easily added in front of the style definition above.

Box Positioning

Since divs are block elements (as opposed to inline), they take up the entire width of the page by default.

Adding the rule display:flex; to the appropriate CSS will override this behavior and let divs appear next to each other.

Document Object Model

DOM

Document Object Model (DOM) is a language-independent convention for representing and interacting with objects in HTML and other languages.

Markup language documents are organized in a tree structure, called the DOM tree.

5. HTTP

The Hypertext Transfer Protocol (HTTP) is an application protocol for distributed, collaborative, hypermedia information systems. HTTP is the foundation of data communication for the World Wide Web.

The internet uses HTTP to communicate with you, your browser and servers.
